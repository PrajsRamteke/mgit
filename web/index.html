Installation

Get up and running
in under a minute.

Choose your preferred installation method below. The quick install is recommended for most users.

Quick Install

Recommended

Copy each line and paste in your terminal one by one:

curl -fsSL https://raw.githubusercontent.com/PrajsRamteke/mgit/main/install.sh | bash echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc source ~/.zshrc mgit --version mgit --help

Install with pipx

Alternative

If you prefer using pipx for isolated installs:

# Install pipx (if needed) brew install pipx pipx ensurepath # Install mgit pipx install git+https://github.com/PrajsRamteke/mgit.git

Verify Installation

mgit --version mgit --help
Quick Start

Five steps to full
multi-account setup.

Follow these steps to add accounts, configure SSH keys, and start switching between identities.

Add Your Account

+

Just provide your GitHub/GitLab username — mgit automatically fetches your name and email.

# Add GitHub account (set as default) mgit add YourGitHubUsername -d # Add with custom profile name mgit add YourGitHubUsername -n work # Add GitLab account mgit add YourGitLabUsername -p gitlab # Add Bitbucket account mgit add YourBitbucketUsername -p bitbucket

Example Output

i Fetching user details from github...
  Name: Your Name
  Email: your@email.com
✓ Profile 'YourGitHubUsername' created successfully!

Get Your SSH Public Key

+

Copy your public key to add it to your Git hosting provider:

mgit key YourProfileName

Output

Public key for 'YourProfileName':

ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIG... your@email.com

Copy the entire key — you will need it in the next step.

Add SSH Key to Provider

+

Add your SSH public key to the appropriate platform:

GitHub

Go to SSH Settings, click "New SSH key", paste and save.

GitLab

Go to SSH Keys, paste your key and save.

Bitbucket

Navigate to Personal Settings, then SSH keys, and add your key.

Test Connection

+

Verify that your SSH connection works:

mgit test YourProfileName

Expected Output

✓ SSH connection successful for 'YourProfileName'
  Hi YourProfileName! You've successfully authenticated.

Start Using

+

Switch between profiles and clone with specific identities:

# Switch to a profile (global) mgit use YourProfileName # Switch for current repository only mgit use YourProfileName -l # Clone with specific profile mgit clone YourProfileName git@github.com:username/repo.git

Advanced

Configuration options
and power features.

Fine-tune mgit for complex setups and advanced requirements.

Add Command Options

mgit add <username> [OPTIONS]
  • -p, --provider Git provider: github, gitlab, bitbucket, custom (default: github)
  • -n, --name Profile name (default: username)
  • -e, --email Override email (auto-fetched if not provided)
  • -d, --default Set as default profile
  • -w, --workspace Workspace directory for auto-switching
  • -k, --key-type SSH key type: ed25519, rsa (default: ed25519)
  • --passphrase SSH key passphrase
  • --signing-key GPG signing key ID
  • --custom-host Custom hostname (required when provider=custom)

Workspace Auto-Switching

Automatically switch profiles based on the directory you are working in. Set a workspace once and mgit handles the rest.

mgit workspace work ~/work mgit workspace personal ~/personal

When you cd into a configured workspace directory, mgit will automatically apply the corresponding profile's Git configuration.

TIP: Combine workspaces with directory-specific .gitconfig includes for maximum flexibility.

Troubleshooting

Common issues
and fixes.

Quick solutions for the most frequently encountered problems.

macOS "externally-managed-environment" error

+

This occurs on newer macOS versions with system Python protections. Use one of these solutions:

# Use the installer script (recommended) curl -fsSL https://raw.githubusercontent.com/PrajsRamteke/mgit/main/install.sh | bash # Or use pipx brew install pipx && pipx ensurepath pipx install git+https://github.com/PrajsRamteke/mgit.git

mgit: command not found

+

The mgit binary is not in your PATH. Add it:

echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc source ~/.zshrc

If you use bash instead of zsh, replace ~/.zshrc with ~/.bashrc.

SSH connection failed

+

If the SSH connection test fails, check the following:

  1. Verify that the SSH key was added to GitHub/GitLab/Bitbucket.
  2. Check that the key matches: mgit key <name>
  3. Start the SSH agent: eval "$(ssh-agent -s)"

Permission denied (publickey)

+

Try testing and manually adding the key to the SSH agent:

mgit test YourProfileName ssh-add ~/.ssh/id_ed25519_YourProfileName

Email not showing / not public

+

If your email is not set to public on GitHub, mgit will automatically use the GitHub-provided noreply email address. No action needed in most cases.

To override manually, use the -e flag when adding a profile:

mgit add YourUsername -e your@email.com

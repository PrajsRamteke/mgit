Installation

Get up and running
in under a minute.

Choose your preferred installation method. Copy one by one and paste in terminal.

Quick Install

Recommended

The fastest way to get started. Uses the install script and sets up your PATH automatically.

bash 
curl -fsSL https://raw.githubusercontent.com/PrajsRamteke/mgit/main/install.sh | bash
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
mgit --version
mgit --help

Install with pipx

Alternative

If you prefer using pipx for isolated Python tool installations. Great for keeping your system clean.

bash 
# Install pipx (if needed)
brew install pipx
pipx ensurepath

# Install mgit
pipx install git+https://github.com/PrajsRamteke/mgit.git

Verify installation:

bash 
mgit --version
mgit --help
Quick Start

Five steps to
multi-account mastery.

Follow this guided workflow to add your accounts, configure SSH, and start working across multiple Git identities.

01

Add Your Account

Just provide your GitHub/GitLab username and mgit automatically fetches your name and email.

bash 
# Add GitHub account (set as default)
mgit add YourGitHubUsername -d

# Add with custom profile name
mgit add YourGitHubUsername -n work

# Add GitLab account
mgit add YourGitLabUsername -p gitlab

# Add Bitbucket account
mgit add YourBitbucketUsername -p bitbucket
Expected Output: Fetching user details from github...
  Name: Your Name
  Email: your@email.com
Profile 'YourGitHubUsername' created successfully!
02

Get Your SSH Public Key

Copy your public key to add it to your Git provider settings.

bash 
mgit key YourProfileName
Output: Public key for 'YourProfileName':

ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIG... your@email.com

Copy the entire key -- you will need it in the next step.

03

Add SSH Key to Your Provider

Paste your SSH public key into your Git provider's settings.

  • 1 Go to GitHub SSH Settings
  • 2 Click "New SSH key"
  • 3 Paste your key and save
  • 1 Go to GitLab SSH Keys
  • 2 Click "Add new key"
  • 3 Paste your key and save
  • 1 Go to Bitbucket → Personal Settings
  • 2 Navigate to SSH keys
  • 3 Add your key and save
04

Test Connection

Verify that your SSH key is properly configured and can authenticate with your provider.

bash 
mgit test YourProfileName
Expected Output: SSH connection successful for 'YourProfileName'
  Hi YourProfileName! You've successfully authenticated.
05

Start Using

Switch between profiles, clone repositories, and work with your preferred identity.

bash 
# Switch to a profile (global)
mgit use YourProfileName

# Switch for current repository only
mgit use YourProfileName -l

# Clone with specific profile
mgit clone YourProfileName git@github.com:username/repo.git

Examples

Real-world usage
patterns.

Common workflows and how to accomplish them with mgit.

Add Multiple Accounts

Set up personal, work, and freelance accounts with different providers.

bash 
# Personal GitHub (default)
mgit add johndoe -d

# Work GitHub
mgit add john-doe-company -n work

# GitLab account
mgit add jdoe -p gitlab -n freelance

View All Accounts

List all configured profiles with their providers and status.

bash 
mgit ls

Check current configuration:

bash 
mgit current

Preview User Info

Look up user details from any provider before adding a profile.

bash 
mgit info SomeUsername
mgit info SomeUser -p gitlab
Advanced

Advanced options &
configuration.

Fine-tune your profiles with extended options and workspace-based auto-switching.

>

Add Command Options

usage 
mgit add <username> [OPTIONS]
  • -p, --providerGit provider: github, gitlab, bitbucket, custom (default: github)
  • -n, --nameProfile name (default: username)
  • -e, --emailOverride email (auto-fetched if not provided)
  • -d, --defaultSet as default profile
  • -w, --workspaceWorkspace directory for auto-switching
  • -k, --key-typeSSH key type: ed25519, rsa (default: ed25519)
  • --passphraseSSH key passphrase
  • --signing-keyGPG signing key ID
  • --custom-hostCustom hostname (required when provider=custom)
#

Workspace Auto-Switching

Automatically switch profiles based on your working directory. Associate a profile with a directory and mgit handles the rest.

bash 
mgit workspace work ~/work
mgit workspace personal ~/personal

When you navigate into ~/work, mgit will automatically use your "work" profile. Navigate to ~/personal and it switches to "personal". No manual switching needed.

Troubleshooting

Common issues
and fixes.

Quick solutions to the most frequently encountered problems.

This happens when macOS prevents pip from installing packages globally. Use the installer script or pipx instead.

bash 
# Use the installer script (recommended)
curl -fsSL https://raw.githubusercontent.com/PrajsRamteke/mgit/main/install.sh | bash

# Or use pipx
brew install pipx && pipx ensurepath
pipx install git+https://github.com/PrajsRamteke/mgit.git

Your PATH may not include the installation directory. Add it manually:

bash 
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

If your SSH connection test fails, follow these steps in order:

  • 1 Verify SSH key was added to GitHub/GitLab settings
  • 2 Check the key matches: mgit key <name>
  • 3 Start SSH agent: eval "$(ssh-agent -s)"

Ensure your SSH key is added to the agent. Test and add it manually:

bash 
mgit test YourProfileName
ssh-add ~/.ssh/id_ed25519_YourProfileName

If your email is not publicly visible on your Git provider, mgit will automatically use the provider's noreply email address (e.g., GitHub's noreply format). You can also manually specify an email during profile creation:

bash 
mgit add YourUsername -e your@email.com
